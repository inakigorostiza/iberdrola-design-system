# Presentation template — Iberdrola Design System

A self-contained, on-brand **16:9 slide deck** built entirely with the Iberdrola
Design System. Open `index.html` in any modern browser — no build step, no
dependencies beyond the design-system stylesheet (`../../css/iberdrola.css`).

It is the presentation **master**: ten ready-made slide layouts you duplicate and
edit to assemble a talk.

---

## How to present

Open `index.html` and press <kbd>F</kbd> for fullscreen. Each slide is a true
16:9 frame, letter-boxed on screens that aren't exactly 16:9.

| Action | Keys |
| --- | --- |
| Next slide | <kbd>→</kbd> · <kbd>↓</kbd> · <kbd>Space</kbd> · <kbd>PageDown</kbd> |
| Previous slide | <kbd>←</kbd> · <kbd>↑</kbd> · <kbd>PageUp</kbd> |
| First / last slide | <kbd>Home</kbd> / <kbd>End</kbd> |
| Toggle fullscreen | <kbd>F</kbd> |
| Advance / go back | Click the **right** / **left** third of a slide |

You can also just **scroll** — slides snap into place. A thin tricolor progress
bar (top) and a per-slide footer (Iberdrola icon + `NN / 10`) always show where
you are.

**Deep-linking:** every slide has an id (`#s1` … `#s10`). Open
`index.html#s7` to jump straight to the chart slide — handy for demos and links.

## How to duplicate a slide

1. Each slide is one `<section class="slide">` block in `index.html`. Copy the
   whole block (from `<section …>` to `</section>`) and paste it where you want
   the new slide.
2. Give it a fresh `id` (e.g. `id="s11"`) or remove the `id` if you don't need a
   deep link.
3. Edit the text/values inside. **You don't have to renumber anything** — the
   footer counter (`NN / 10`) and progress bar are generated by JavaScript from
   the number of slides on the page.
4. Pick a background by swapping the frame modifier class:
   `slide__frame--forest` · `--tint-mint` · `--tint-peach` · `--tint-sky` ·
   `--tint-sand` · `--subtle`, or none for plain white.

**Golden rule:** compose from design-system classes and tokens. No hard-coded
colours or sizes — use `--ib-*` custom properties and `.ib-*` component classes
so every slide stays on-brand. Deep-forest (`--forest`) slides automatically use
white text.

### Reusable pieces you'll want
- `ib-display`, `sl-headline`, `ib-lead`, `ib-overline` — type roles
- `ib-badge` / `ib-badge--solid` / `--info` / `--accent` — pills
- `ib-stat` (`__value` / `__label` / `__desc`) — big figures
- `ib-card` — rounded content card
- `ib-ph` / `ib-ph--photo` / `ib-ph--cover` — image placeholders (swap for a real
  `<img>` in production)
- `ib-energy-line`, `ib-text-gradient` — tricolor flame accents
- `sl-list` — bullets with tricolor markers · `sl-chart` — the CSS bar chart

## Exporting to PDF (for sharing)

The deck has a print stylesheet: each slide becomes one page.

1. In the browser, **File → Print** (or <kbd>Cmd/Ctrl</kbd> + <kbd>P</kbd>).
2. Destination: **Save as PDF**.
3. Layout: **Landscape**. Paper: **16:9 / widescreen** if offered (otherwise A4
   landscape is fine). Margins: **None**. Enable **Background graphics**.
4. Save. You get a clean, one-slide-per-page PDF to email or attach.

## Design-system token → PowerPoint / Google Slides equivalent

Use this when rebuilding the deck (or a matching one) in PowerPoint or Google
Slides so it stays on-brand.

### Slide setup
| Setting | Value |
| --- | --- |
| Slide size | **Widescreen 16:9** (PowerPoint: Design → Slide Size → Widescreen; Google Slides: File → Page setup → Widescreen 16:9) |
| Body / heading font | **Iberdrola** (brand) — fall back to **Figtree** where Iberdrola isn't installed; then Segoe UI / system sans |
| Share format | Export to **PDF via the browser Print dialog** (see above) for pixel-accurate sharing |

### Gradients & motifs
| Token | Value | Use |
| --- | --- | --- |
| `--ib-gradient-forest` | `linear-gradient(160deg, #006B33 0%, #013D28 100%)` | Title / quote / closing backgrounds |
| `--ib-gradient-tricolor` | `linear-gradient(120deg, #00A443, #0DA9FF 55%, #FF9C1A)` | "Energy line" rule, gradient numerals, highlighted text |

### Type scale (approx. point sizes on a 16:9 slide)
| Token | Web size | ≈ PowerPoint pt |
| --- | --- | --- |
| `--ib-text-display` | up to 4.5rem | ~54–60 pt |
| `--ib-text-h1` | up to 3.25rem | ~40 pt |
| `--ib-text-h2` | up to 2.5rem | ~30 pt |
| `--ib-text-body-lg` | 1.125rem | ~18 pt |
| `--ib-text-body` | 1rem | ~16 pt |
| `--ib-text-caption` / overline | 0.75rem | ~11 pt (overline: uppercase, letter-spacing) |

### Shape / spacing conventions
- **Buttons & badges:** fully rounded "pill" shape (`--ib-radius-pill`).
- **Cards:** 16 px corner radius (`--ib-radius-lg`), soft green-tinted shadow.
- **Spacing:** multiples of 8 px (`--ib-space-2` = 8, `--ib-space-5` = 24,
  `--ib-space-7` = 48…). Keep generous whitespace.
